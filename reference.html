Color tokens

Five core CSS custom properties drive almost everything. Dark mode swaps the entire set in one rule. Anything keyed to var(--ink), var(--muted), etc. follows automatically.

Severity tokens

Seven tokens drive every severity color in the codebase: .q-* tier classes, .alert-* pill backgrounds, kpColor / bzColor / speedColor JS helpers, the bootstrap warning, and the dscovr Pdyn / xray / sym-H sparklines. Four primary tiers (excellent / good / warn / bad) get dark-mode variants; three secondary tokens (strong / extreme / info) reuse the same hex in both modes.

--sev-excellent
.q-excellent / verdict tier 5
--sev-good
.q-good / kp<active / bz>0 / speed<400
--sev-warn
.q-warn / .alert-watch / pdyn≥5
--sev-strong
kp G1-G2 / bz storm-severe
--sev-bad
.q-bad / .alert-alert / .bootstrap-warn
--sev-extreme
.alert-extreme only
--sev-info
.alert-info pill background

.q-muted reuses var(--muted) directly; not a dedicated severity token. Toggle the OS dark-mode preference to see the four primary tokens shift; --sev-strong / --sev-extreme / --sev-info stay constant pending dedicated dark tuning.

Captions (muted-italic family)

Five class names share one visual rule (11px muted italic), but they encode four distinct semantic roles. Listed here so the overlap is visible.

.panel-caption - "Courtesy of NASA/SDO and the AIA, EVE, and HMI science teams."

role: source / license attribution. Margin 4px / 14px.

.pending-note - "Loading band data..."

role: loading state placeholder. Margin 4px / 8px.

.empty-list - "No active entries."

role: empty data state. Same rule, different name.

.iono-note - "ARTIST-5 autoscale (GIRO). Data provider: ..."

role: panel footnote. 10px (smaller than the rest), margin 6px / 0.

.simple-footer - "Live derivation - fetched 26-05-06 18:42 UTC"

role: glance table attribution / freshness. max-width 640px.

.freshness-note - "fetched 26-05-06 18:42 UTC" - belongs to a different family (tabular, not italic).

role: refresh timestamp on data panels. 10px tabular muted, NOT italic.
