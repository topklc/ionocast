Data source licenses and attribution

Every data source, image, and embedded widget used by ionocast, with its license, required attribution string, and any constraints that affect how the site can be operated.

Operational status of ionocast: free, personal / hobby amateur-radio dashboard. No ads. No paywalls. No paid tiers. No commercial use. The footer carries a Ko-fi link for voluntary donations toward hosting costs: nothing is gated behind a donation, no rewards are offered, and every panel remains free for every visitor regardless of whether they donate. Five of the sources below forbid commercial use; any future ads, paywall, or paid tier would break all five simultaneously. See Commercial-use ban at the bottom.

Last audited: 2026-04-26.

1. NOAA SWPC - services.swpc.noaa.gov

Used for: 3-day forecast, 27-day outlook, solar regions, D-RAP grid, OVATION aurora, planetary Kp, F10.7 flux, GOES X-ray / protons / electrons, DSCOVR solar-wind mag + plasma, active alerts, animation frames.

  • License: Public domain (17 USC § 105).
  • Policy: weather.gov/disclaimer, swpc.noaa.gov/content/data-access
  • Personal use: yes. Commercial use: yes.
  • Attribution required: no. Best-practice credit: Data: NOAA/SWPC.
  • Rate limits: no published hard limit. NWS monitors for abuse and can block IPs. Guidance: poll no faster than the product's refresh cadence (e.g. 60 s for 1-min products), back off 60 s on errors.
  • Trademarks: NOAA / NWS names and logos are trademark-protected. Do not imply official endorsement.

2. NASA CCMC DONKI - kauai.ccmc.gsfc.nasa.gov/DONKI

Used for: CME catalog (CMEAnalysis) and coronal-hole high-speed stream catalog (HSS).

  • License: Public domain (17 USC § 105). Data is labelled "experimental research", not an operational product.
  • Policy: ccmc.gsfc.nasa.gov/data-policy
  • Personal use: yes. Commercial use: yes.
  • Attribution required: no. Best-practice credit: Data courtesy of NASA CCMC DONKI.
  • Rate limits: not published. Keep polling conservative; DONKI is a lightly-resourced research service.
  • Labelling duty: surface somewhere near the panel that this is preliminary NASA research data, not operational.

3. Andrew Rodland KC2G - prop.kc2g.com

Used for: real-time ionosphere station JSON (/api/stations.json) and the foF2 SVG render.

  • License: not stated on the site. The site's code is open-source; the data itself is derived from GIRO ionosondes and therefore inherits GIRO's CC-BY-NC-SA 4.0 terms.
  • Policy / acknowledgments: prop.kc2g.com/acknowledgments
  • Personal / non-commercial: yes. Commercial use: no (NC flow-through from GIRO).
  • Attribution required: yes (expected). Credit: Ionospheric map by Andrew Rodland (KC2G), prop.kc2g.com, plus pass through GIRO's station-operator credits.
  • Rate limits: not published; 5 min edge cache is considerate.

4. wspr.live - db1.wspr.live

Used for: ClickHouse API calls: per-band aggregate counts for the last hour, and parameterized historical spot counts for prediction scoring.

  • License / terms: wspr.live. "for your own research and projects, as long as the results are accessible free of charge for everyone." Commercial / profit-oriented use explicitly prohibited.
  • Personal / non-commercial: yes. Commercial use: no.
  • Attribution required: yes (expected). Credit: WSPR data: wspr.live (sourced from wsprnet.org).
  • Rate limits: hard cap of 20 requests / minute (since Oct 2023). Burst briefly tolerated. GET only; only ?query= accepted. Headers filtered, POST blocked.
  • Constraint: the "free of charge for everyone" clause means if ionocast ever adds ads, paywalls, or any access restriction to WSPR panels, this license breaks.

  • License / terms: the IMO publishes the annual shower calendar as public reference material for amateur observers. No formal redistribution license; attribution is the convention across the meteor-observing community.

Voluntary donations

The Ko-fi link in the footer is treated as non-commercial under each of the licenses above:

  • Donations are unsolicited beyond the link itself; no rewards, no perks, no tiers.
  • No content is gated; every panel is free to every visitor whether or not they donate.
  • Ko-fi proceeds offset hosting and domain costs; ionocast is not a business and has no revenue model.

This follows the standard Creative Commons reading of the NC clause: the test is "primarily intended for or directed toward commercial advantage or monetary compensation," and a tip jar with no consideration in return does not meet it. See the CC NC FAQ for the canonical interpretation.

If any individual upstream operator reads "non-commercial" or "personal hobby use" more strictly than this and objects to the donate link, the response is to drop the affected panel, not the donate link. The audit trail of which upstream operators have been contacted and what they replied is kept in docs/UPSTREAM-CONTACTS.md in the project repository.

Source code

The full ionocast implementation - browser code, Cloudflare Pages Functions for the upstream proxies, the offline calibration harness, and the LaTeX whitepaper - is open at github.com/topklc/ionocast.

The site has no third-party JavaScript libraries, no build step, and no bundler. Everything served to the browser is plain HTML, ES modules, and CSS. There is no telemetry, no analytics, and no tracking.

Privacy

ionocast keeps operator data on the device wherever possible. The QTH is one exception: two panels send it to ionocast's own proxies for nearest-station lookups, detailed below. Beyond that, no location, settings, or telemetry leaves the device. Specifically:

  • The QTH (Maidenhead grid square) is stored in the browser's localStorage and read only by the page's own JavaScript. The Ionosphere and VHF tropospheric panels send the QTH to ionocast's /api/giro and /api/tropo Cloudflare Pages Functions so each can find the nearest digisonde / radiosonde station. The proxy uses the QTH only to compute distances and never persists or forwards it to the upstream service. Cloudflare's edge logs may briefly capture the request URL per their standard privacy policy.
  • Operator settings (transmit power, antenna type, antenna gain, antenna height, mode, noise environment) live in the same localStorage and never leave the device.
  • Theme preference (light / dark / auto) and selected language live in localStorage.
  • WSPR queries to db1.wspr.live are global hourly aggregates with no grid field; the operator's location is never transmitted to the WSPR service.
  • Upstream queries to giro.uml.edu and weather.uwyo.edu identify the nearest station by code only; the QTH-to-code translation happens on the ionocast proxy, so the upstream services never receive QTH coordinates.

No cookies. No analytics. No third-party tracking pixels. No advertising networks. The page does not register a service worker.

Cloudflare Pages serves the static files and proxies the upstream API requests through edge functions. Cloudflare's privacy and logging policies apply to that layer; ionocast does not add its own logging on top.
